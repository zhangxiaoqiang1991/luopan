# 受众视角确认

## 核心原则

先确认用途，再开始研究。用户没有说明投资或求职目的时，不得自动猜测，不得抓取后再决定，也不得默认生成投资与求职并重的报告。

## 第一步：识别已有意图

用户已经出现以下信号时直接开始，不重复询问：

- 买入、持仓、估值、股价、安全边际、回报率：`investment`；
- 投递、面试、岗位、团队、薪酬、Offer：`career`；
- 明确要求两者，或同时提出交易与岗位问题：`balanced`。

## 第二步：用途不明时必须三选一

用户只说“研究一下某公司”“这家公司怎么样”或类似表达，且上下文没有用途线索时，在任何深度研究前询问：

> 你这次调研主要想解决哪类问题？
>
> 1. 投资为主——重点看财务、护城河、估值和安全边际
> 2. 求职为主——重点看业务前景、组织稳定性、岗位价值和面试反问
> 3. 投资与求职并重——两条主线都看，但会共享事实、避免重复

只问这一道，不要求用户复述商业模式、收入模式或关键变量。等待用户选择后再研究，不得为了显得主动而先生成一份可能需要推倒重来的报告。

如果用户说“直接做”“不要问”，应请用户至少回复 `1 / 2 / 3`；这不是问卷，而是决定报告结构的必要输入。只有用户的原始请求本身已经包含用途，才可以不问。

## 第三步：按选择控制信息量

### 投资为主

- 完整展开财务、商业模式、护城河、治理、估值、安全边际和反证；
- 不生成求职章节和面试10问；
- 如用户后续提出求职问题，再补充求职模块。

### 求职为主

- 完整展开公司稳定性、业务位置、岗位价值、上级与组织、职业资本和面试10问；
- 上市公司财务只保留与公司和业务稳定性有关的部分，不展开估值底稿；
- 不输出买入价格或投资建议。

### 投资与求职并重

- 共享公司身份、财务、商业模式和生态位事实，避免重复；
- 投资与求职分别形成独立结论，不合成总分；
- 两条主线都进入主阅读路径，但仍使用折叠层控制长度。

## 中国用户场景

公司国籍可以影响资料路线和问题相关性，但不能代替用户选择：

- 国内公司可能同时具有投资与求职价值，仍然询问；
- 国外上市公司可能更常用于投资研究，仍然询问；
- 国外非上市公司可能更适合商业研究。若用户三项都不适用，可以接受“只看公司与商业模式”的自然语言回答，并路由为一般公司研究；
- 不把“国外公司没有求职价值”写成绝对规则，因为中国区、远程和海外岗位可能存在。

## 记录方式

将用户选择写入 `presentation.primary_audience`：`investment`、`career` 或 `balanced`。`presentation.reason` 应记录“用户选择”，而不是 Agent 对公司受众的猜测。
