# 理解测试卡片

## 何时读取

生成 HTML 与 Markdown 报告的理解测试时读取。

## 硬性规则

1. 测试卡片必须是**选择题**，不使用填空题、简答题或要求用户复述。
2. 每题明确标记 `single_choice` 或 `multiple_choice`。单选题只有一个最佳答案；多选题至少有两个正确选项，并提示“可多选”。
3. 每题提供 3–5 个选项。干扰项应代表真实常见误解，不得用明显荒谬的选项凑数。
4. 每题必须包含：正确答案、答案解析、每个关键选项为何对/错、关联报告章节和证据 ID/来源链接。
5. 只根据报告已有证据出题，不添加新事实。正确答案不得与正文结论或数据口径冲突。
6. 测试是报告后的可选学习增强，不是使用 Skill 的前置门槛。
7. 默认不输出精确的“理解得分”或能力排名。用户选择后显示对/错、解析和相关追问。

## 出题范围

默认生成 4–6 题，优先覆盖：

- 公司如何创造价值和赚钱；
- 增长来源与增长质量；
- 护城河的真实证据与趋势；
- 最大风险与可能推翻结论的信号；
- “好公司”与“当前好投资”的区别；
- “好公司”与“具体好岗位”的区别。

不优先测试孤立数字记忆。数字题应检查趋势、口径或因果，而不是死记某年营收。

## HTML 行为

- 未选择前不显示正确答案。
- 单选使用 radio，多选使用 checkbox。
- 用户提交后显示正确状态、解析、证据链接和“复制追问”。
- 不收集、上传或持久化用户答案，除非用户明确要求并同意。

## Markdown 行为

- 显示题目、题型和选项。
- 将正确答案、解析和证据放在 `<details>` 折叠区。
- 即使阅读器不支持折叠，内容仍必须保持可读。
